## Safest External-Skill Sequence

1. Keep the new local OpenStem repo-scoped skills as the active guardrails.
2. Run SkillSpector static scans on any external agent skill before install or copy.
3. Manually review hooks, commands, install scripts, and license files for scanned skills.
4. If approved, try `shadcn/improve` in read-only planning mode first.
5. Consider Ponytail only in conservative/default mode and only if it preserves OpenStem validation and honesty rules.
6. Add drawio-skill only for docs diagrams, never as an OpenStem runtime dependency.
7. Keep Handy as a reference project only.
