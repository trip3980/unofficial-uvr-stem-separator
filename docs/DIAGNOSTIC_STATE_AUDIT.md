## Diagnostic Code System

Structured diagnostic definitions live in `src/services/diagnosticCodes.ts`.

Each diagnostic entry includes:

- `code`
- `severity`
- `category`
- `userMessage`
- `developerMessage`
- `suggestedFix`
- `proofImpact`
- `canUserFix`
- `canDeveloperFix`
- `expectedBlocker`

## Expected Blockers

These are correct gates and should remain blocked:

- `PROOF_BETA_BLOCKED`: Beta Candidate is not approved unless proof evidence and final release review both pass.
- `PROOF_MODEL_MISSING`: no verified single model weight is proof-ready.
- `PROOF_MODEL_HASH_MISSING`: expected SHA-256 is missing.
- `MODEL_LOCAL_HASH_MISMATCH`: local model hash mismatch.
- `MODEL_SOURCE_AUTH_REQUIRED`: Hugging Face/source access requires authorization.
- `MODEL_SOURCE_BROKEN_LINK`: configured source returns HTTP 404.
- `RUNTIME_BROWSER_PREVIEW_ONLY`: browser preview cannot run native IPC workflows.
- `PROOF_FFMPEG_FALLBACK_NON_AI`: FFmpeg fallback is non-AI.
- `MIXER_NO_VERIFIED_SESSION`: mixer has no verified real stems.
- `ENSEMBLE_PLANNER_ONLY` and `ENSEMBLE_BACKEND_NOT_IMPLEMENTED`: ensemble execution is not implemented.
- `BASIC_PITCH_DRY_RUN_ONLY`: browser Basic Pitch cannot write local MIDI files.
- `GENERATIVE_CONNECTOR_NOT_CONFIGURED`: no supported user-authorized generative connector is configured.
- `YUE_LOCAL_ENGINE_NOT_WIRED`: YuE local generation lacks wired local weights/backend proof.

## Principle

Blocked is not automatically bad. A blocker is healthy when it preserves truth, names the owning module, exposes a stable code, and tells the user or developer what must change before the action can run.
