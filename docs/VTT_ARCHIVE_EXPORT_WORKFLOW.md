# VTT Archive and Export Workflow

OpenStem supports a local-first VTT intake plan for Zoom-style or similar transcript files. This workflow helps users avoid copy/paste and print-to-PDF workarounds while keeping export honesty strict.

Release state remains Hardened Functional Alpha. This workflow is not AI stem-separation proof and does not approve Beta Candidate.

## Required Flow

1. Import VTT from a visible source folder.
2. Parse transcript locally.
3. Rename session or accept the default name.
4. Rename detected speaker labels if needed.
5. Preview the renamed transcript.
6. Click Archive & Export when native writer support exists.
7. Write renamed transcript archive.
8. Write selected export formats.
9. Verify every output file by path and nonzero size.
10. Add a local history entry only after verified output exists.

In Automatic then Review mode, OpenStem may parse and prepare the archive/export plan first, then keep title, session number, speaker labels, transcript text, export format, and destination folder editable for later cleanup.

In checkpoint automation, imported VTT can start at Source Intake, skip audio-only stages, stop at Transcript Preview, continue through Speaker Review and Title / Filename Review, then rerun only Transcript Archive Saved or Transcript Export Saved after edits. Failed archive/export stages must preserve the parsed VTT, speaker map, title metadata, and previous verified outputs.

## VTT Parsing

The first OpenStem parser slice is implemented in `src/services/vttTranscriptImport.ts`.

It must:

- require a `WEBVTT` signature,
- parse cue timestamps,
- preserve cue order,
- preserve source path metadata when provided,
- detect simple speaker labels,
- convert to clean transcript text,
- return structured errors for malformed or empty input.

States:

- `VTT_FILE_MISSING`
- `VTT_PARSE_READY`
- `VTT_PARSE_FAILED`
- `VTT_IMPORTED`
- `VTT_SPEAKERS_DETECTED`
- `VTT_NO_SPEAKERS_DETECTED`
- `VTT_ARCHIVE_READY`
- `VTT_ARCHIVE_WRITTEN`
- `VTT_ARCHIVE_FAILED`

## Speaker Rename

Speaker rename is label editing only.

Rules:

- do not claim diarization,
- do not invent speakers,
- preserve original labels in metadata,
- allow reset,
- do not overwrite source VTT,
- do not export until a safe filename and output folder are selected.

States:

- `SPEAKER_LABELS_DETECTED`
- `SPEAKER_RENAME_READY`
- `SPEAKER_RENAME_APPLIED`
- `SPEAKER_RENAME_RESET`
- `SPEAKER_LABELS_NOT_FOUND`

## Folder Policy

Separate source folders from archive/export folders:

- VTT source folder: `{userData}/OpenStem/Transcription/Imported VTT Transcripts`
- renamed archive folder: `{userData}/OpenStem/Transcription/Renamed Transcript Archive`
- PDF export folder: `{userData}/OpenStem/Transcription/Exports/PDF`
- DOCX export folder: `{userData}/OpenStem/Transcription/Exports/DOCX`
- TXT/JSON/SRT/VTT export folder: `{userData}/OpenStem/Transcription/Exports/TXT-JSON-SRT-VTT`

Do not hardcode user folders.

## Auto-Export Options

Default:

- Auto export after rename: off
- Auto export after VTT import: off until native writers are wired
- Auto-create history record: metadata preview only until verified output exists
- Keep original VTT: on
- Overwrite original VTT: off
- Delete source files: off

Possible future options:

- Export PDF after rename
- Export DOCX after rename
- Export TXT after rename
- Export JSON metadata after rename
- Write renamed VTT copy

All options require native file-write verification before success.

## Export Formats

TXT:

- Safest first native writer.
- Writes clean renamed transcript text.
- Requires output file existence and nonzero size.

JSON:

- Stores metadata, segments, and speaker map.
- Must not include private source paths unless the user chooses that behavior.

VTT:

- Writes a renamed VTT copy.
- Must preserve timestamp order.
- Must not overwrite the imported source VTT by default.

SRT:

- Planned until conversion is implemented.
- Requires timestamped segments.

PDF:

- Planned / Not active.
- Must be generated by a local exporter.
- Completion requires file exists, nonzero size, and expected path match.

DOCX:

- Planned / Not active.
- Requires a stable local exporter and license review.
- Completion requires file exists, nonzero size, and expected path match.

ODT/RTF/HTML:

- ODT is converter-gated unless a stable local ODT writer exists.
- RTF remains planned until a local writer exists.
- HTML output must be escaped or sanitized and verified as a real file.

The broader document format policy is defined in `docs/DOCUMENT_FORMAT_WORKFLOW.md` and `src/services/documentFormatPolicy.ts`. Format visibility is not output completion.

## Verification

Export success requires:

- native writer returned success,
- expected path exists,
- file size is greater than 0,
- source VTT was not overwritten unless user explicitly chose that policy.

Native Electron mode is required before OpenStem can write or verify renamed archives, TXT/JSON/VTT exports, PDF files, or DOCX files.

Regenerated exports follow the same verification rule. A filename preview, editor save, or overwrite policy does not prove that an export exists.

Checkpoint automation uses `Output Not Verified` for every archive/export row until the native writer verifies a real nonzero file. Browser preview cannot mark PDF, DOCX, TXT, JSON, SRT, or VTT output complete.

Failure states:

- `ARCHIVE_EXPORT_FAILED`
- `ARCHIVE_EXPORT_PARTIAL`
- `EXPORT_OUTPUT_NOT_VERIFIED`
- `EXPORT_OUTPUT_ZERO_BYTES`

## IPC Plan

Future safe IPC should expose narrow calls:

- `selectVttFiles()`
- `selectVttFolder()`
- `selectArchiveFolder()`
- `selectTranscriptExportFolder()`
- `parseVttTranscript(path)`
- `writeRenamedTranscriptArchive(request)`
- `exportTranscript(request)`
- `verifyTranscriptExport(path)`

Main process rules:

- validate extensions,
- validate selected folders,
- write only inside selected/default archive folders,
- no arbitrary command execution,
- no raw transcript logging,
- no cloud upload,
- return structured success/error states.
